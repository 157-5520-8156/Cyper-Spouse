# 世界模式连续对话体验修复闭环

更新时间：2026-07-12
状态：执行中
优先级：P0，上线阻断
范围：世界模式下的连续对话理解、用户事实、角色事实表达、通信动作与逻辑时钟并发
不在范围：小屋界面与素材、自由 NPC LLM、单纯文风润色

## 1. 目标

世界账本已经能够推进活动、需求、目标、NPC 关系和经历；本闭环解决的是这些状态进入真实聊天后仍然表现为失忆、复读、答非所问或无依据补写的问题。

完成后，以下链路必须同时成立：

```text
用户消息
→ 识别说话主体、问题目标与时间范围
→ 读取当前对话事实、已确认用户事实和角色世界事实
→ 生成有连续性的候选回复
→ 按声明而非整段执行来源校验
→ 针对失败片段修复或诚实降级
→ 形成可结算通信 Action
→ 投递与逻辑时钟推进并发安全
```

### 2.2 已证实的代码事实

1. 世界路径 `_handle_world_message` 只向模型提供当前用户消息、世界事实、经历、当前场景、自我核心和确认用户画像，没有提供最近已投递对话。
2. 世界路径不会运行旧路径的短期记忆抽取；只有耐久事实才可能进入用户画像，因此“正在赶项目、昨晚没睡、胃不舒服”等回合事实没有稳定入口。
3. 候选校验失败后，只要消息包含“昨天/今天/上午/下午/上次”，就直接选择角色经历作为 grounded fallback，没有先判断问题主体和目标。
4. 最终兜底固定为“嗯，你说。”，无法保持原问题的言语行为。
5. 通信策略只有 active activity 且 energy <= 35 才延迟；本次 active activity、energy=64 被判定为 `world_available`。
6. `advance` 会按逻辑到期时间将所有 scheduled Action 标记过期，未区分尚在真实执行中的 model call。

## 3. 公共测试缝隙

本闭环只在用户/调用方可观察的边界写测试：

1. `WorldKernel`：事实来源、Action 生命周期与确定性回放；
2. `CompanionEngine.handle_message` / `TurnCoordinator`：连续多轮输入到回复或延迟 disposition；
3. HTTP `/messages` 与 `/world/{id}/advance`：幂等、并发、状态码和投递结果。

模型、墙钟和随机数只在外部边界使用可控替身；不 mock 项目内部模块。

## 4. 验收门禁

### 4.1 连续性与主体

- 相邻回合必须向模型提供最近已观察用户消息和已投递角色消息，且当前消息不重复；
- “我今天/昨天为什么……”默认指向用户，不得自动返回角色经历；
- “你今天/昨天做了什么”才允许按对应时间读取角色经历；
- 当前对话中的项目、身体状态、决定和指代至少在会话窗口内可追溯；
- 跨日仍需要的事实必须经过候选/确认规则进入世界事实索引，并保留来源消息 ID。

### 4.2 事实与表达

- 角色的地点、动作、完成经历、NPC 互动和心理原因都需要相应来源或明确使用不确定表达；
- 已提交事实可以自然转述，但不得增加来源不支持的新结果、对话或心理活动；
- 对“谁/为什么/顺利吗”等账本未提供答案的追问，应承认信息有限并围绕已知部分回应，不得复读整条经历冒充答案；
- 校验失败应保留问题类型进行定向修复；不得降级为“嗯，你说”、回显用户问题或无关世界快照；
- 任意回复不得与最近两条外发内容形成无意义复读。

### 4.3 通信动作与并发

- 忙碌判定同时考虑活动 attention demand、interruptible、阶段和用户消息紧急度，不只看低精力；
- 延迟必须只有一个 `reply_later` Action，并能在到期后恰好恢复一次；
- 在途 model call 使用真实执行租约，不因普通逻辑时钟推进直接过期；
- 时钟推进与消息处理并发时，必须得到回复、合法延迟或明确失败终态之一，不能留下 claimed turn 无终态；
- 每条外发仍可追溯至入站消息、intent、模型外部结果、outgoing Action 和投递结算。

### 4.4 最终体验

- 自动化专项、全量测试和 lint 全部通过；
- 新隔离世界完成不少于 30 轮真实模型自适应对话，覆盖同日、跨日、事实追问、情绪话题、纠错、活动中消息和逻辑时间推进；
- 0 条无依据已发生经历；
- 0 条主体错置；
- 0 条无终态入站回合；
- “嗯，你说”式失语、无关快照和逐字复读合计不超过 1 条，且必须有可解释的外部模型失败记录；
- 自然或可接受回复比例不低于 85%。
