# 像素小屋 · Pixel Home

一个**零素材、纯代码**的斜向 2.5D 像素风小屋：所有美术（地板、墙面、家具、墙饰、
角色、动画帧）都由 `js/pixel.js` 的软件光栅化内核在运行时逐像素烘焙成精灵，再以
整数倍最近邻缩放渲染。没有任何图片资产，不依赖 daemon 或编译管线。

美术目标对齐 lofi 风格参考图：黄昏基调 + 多光源暖光（吊灯/台灯/落地灯/串灯/屏幕光）、
紫调环境色、密集软装（植物、挂画、窗帘、地毯花纹、贴纸冰箱、书墙）、家具接触阴影、
拼木地板 + 厨房花砖分区。

## 运行

```bash
open prototypes/pixel-home/index.html                            # file:// 直接可开
python3 -m http.server 8974 --directory prototypes/pixel-home    # 或静态服务器
```

- `index.html?hour=19.8` 指定初始时刻；默认 19:30 黄昏开灯。
- `audit.html` 是美术验收画廊（逐交互特写）；`?debug` 画出角色站位十字线。

## 已实现

- **像素美术**：统一暖色调色板 + 选择性描边；行向拼木地板（逐行色差、板缝、对接缝）、
  厨房花砖分区 + 铜条压边、踢脚线/挂画线、墙面纵向明度渐变、墙基环境光遮蔽。
- **家具**（目录驱动，16 种）：拼布被床、奶油布艺沙发（抱枕+搭毯）、茶几茶具、
  书桌工作位（笔电亮屏、黄铜台灯、绿色办公椅）、餐桌椅（桌旗、花瓶、双人餐位）、
  料理台（灶+水槽+毛巾）、冰箱（磁贴便签）、高书架（书+收纳盒+顶植）、矮书柜、
  衣柜、床头柜（暖灯）、大/小绿植、落地灯、小凳、吊灯。
- **墙面装饰**（数据驱动）：黄昏/白天双版城市窗景 + 褶皱窗帘、吊柜 + 挂具轨、
  瓷砖防溅墙、置物板（书+垂蔓）、挂画、软木便签板、串灯（夜里逐颗发光）、
  吊挂绿植、挂钟、木门。
- **灯光系统**：昼夜色调关键帧（正午透亮 → 日落橙 → 黄昏紫 → 夜蓝）；家具可声明
  `emitters`，夜间渲染径向暖光 + 地面光池 + 呼吸脉动；串灯光点从墙饰烘焙时收集。
- **角色**：四方向行走、眨眼、坐姿（椅背前景层正确遮挡）、侧睡（枕头头 + 沿床轴
  被褥起伏）、手机屏幕辉光；走路影子。
- **引擎**:整数格网 + A\* 寻路；家具占格即碰撞（`decor` 类不占格）；成对遮挡
  判定 + 拓扑排序做深度排序，**角色作为一个排序节点参与**（站在两件家具的
  夹缝里前后遮挡各自正确）；坐椅子按朝向决定人椅盖压方向；接触阴影裁剪在
  地板内；点击地板走位、点击家具互动。
- **自主生活**：作息日程（做饭→吃饭→洗碗→用电脑→睡觉……），手动指令临时接管。
  同一活动有多个提供者时（两盆植物、两把椅子），自动选**离角色最近且可达**
  的一个；手动按钮会把每个实例都列出来。
- **编辑模式**：点选搬动/添加/右键删除家具（选中待放置状态下右键也直接删）、
  **R 键左右翻转**选中或待放置的家具（镜像出两种朝向；四向旋转需要每向出图）、
  冲突红绿预览、布局存 localStorage、导出/导入 JSON。餐桌/书桌的就座点会
  **自动吸附到贴着桌子的椅子**上，椅子拖到桌子哪一侧都能坐对。地毯、墙饰、
  地板分区目前走布局 JSON（`rugs`/`wallDecor`/`zones`）。
- **L 形地板**：布局 JSON 的 `notch: {x,y,w,d}` 从地板外接矩形上切掉一角。
  三种放法：`x=0,y=0` = 切**俯视图左上角**（等距画面的北角。默认布局用
  3x1 的**浅缺口**：厨房所在的左段后墙比右段前移一格，不是大挖角）；
  `y=0` 且贴 `x=w` 一侧 = 切俯视图右上角；含南角（x=w、y=d）= 切底角。
  墙饰 `side:'NE'` 按 t 自动落到对应段，`side:'JOG'` 落在转角短墙上。
  默认分区：厨房左（贴前移矮墙）、床右（窗下）、客厅右下、办公区左下。
  地板烘焙、台座侧边、寻路、灯光裁剪多边形、编辑器放置检查全部跟随。家具
  条目可加 `flip: true` 沿屏幕竖轴镜像：占地 w/d 互换，交互点、朝向、灯光
  挂点自动变换——同一张椅子/沙发精灵借此获得另一侧朝向。默认布局的俯视
  规划图见 `tools/raw/floorplan-asbuilt.png`。

## AI 精灵管线（路线 2）

`assets/ai/manifest.json` 里的家具会用 AI 生成的位图精灵替换程序化烘焙，
缺失的类型自动回退到程序化版本。流程：

1. 以参考图为风格锚生成单件家具：统一 2:1 等距、左上光源、紫调阴影、
   **纯品红 #FF00FF 背景、无地面阴影**，并在提示词里写清占地比例与朝向
   （床头朝右上、沙发开口朝左下、书架格口朝右下）。
2. **标注基准角点**（AI 出图的透视从不精确，必须矫正）：
   先跑 `uv run python tools/fit_base.py --in raw/xxx.png --w 2 --d 3`
   自动拟合底部轮廓两条主边线，输出可直接粘贴的 `--pts`；再用
   `tools/annotate.py --marks` 十字线人工校验（垂蔓、盖毯、脚影会污染
   自动拟合，此时手动改点）。**标注点选"视觉主导结构"的角**——床架横梁、
   沙发底箱、书柜踢脚板——而不是腿尖：肉眼是拿主结构边跟地板缝比的。
   离地的结构角用 `gx,gy,z:px,py` 的 z（HZ 单位）声明高度。
3. `uv run python tools/make_sprite.py --in raw/xxx.png --type bed --w 2 --d 3 \
   --plumb --pts "0,3,0.12:171,670 2,3,0.12:428,810 2,0,0.12:852,575"`
   —— 去背（含品红色相残影）→ **矫正到精确等距基**（过采样）→
   最近邻降采样 → 量化到共享调色板 → 由标注角点**精确推导北角锚点** →
   写 manifest。不给 `--pts` 走旧的按宽缩放路径（不推荐）。
   **务必加 `--plumb`**（点序 W S E）：普通三点仿射会为了让角点精确落位
   而剪切竖线，整件家具看起来"歪的"；`--plumb` 锁死竖线垂直，把两条底边
   斜率精确掰到 ±0.5，角点允许漂几像素（表现为几像素 overhang），观感
   远好于倾斜。桌腿类家具的标注点是腿尖时，网格坐标要按腿的内缩量写
   （如 `0.13,0.87` 而非 `0,1`），否则桌面会被压小。
   校验：`uv run python tools/check_sprite.py` 复测成品底边斜率（目标 ±0.5）
   与占地宽度；`uv run python tools/overlay_check.py` 出线框叠加图
   （红=底面菱形、青=顶面、黄=竖棱），歪斜一眼可见。注意腿/垂坠物会让
   轮廓测量偏差，以叠加图目视为准。
4. **可选前层**：manifest 条目加 `"front": {url, offset}` 可以给精灵配一张
   "镜头侧覆盖层"（如椅子的靠背板，从主图里抠出来存成单独 PNG）。角色坐进
   该家具时按"主体 → 人 → 前层"合成，背对镜头坐时坐垫在人下、靠背压在腿前。
   翻转（flip）时前层跟着镜像。
4. AI 若把家具画得比占地宽（普遍 +10%），可以用**分数格坐标**把主结构
   居中收进碰撞盒，如 `0.12,2:.. 0.88,2:..`（书柜实际按 0.765 格宽矫正），
   视觉压进 1×2 的格子里，通道就不会被精灵压住。
5. 刷新页面即生效；比例/明暗离谱就改提示词重出（提示词里锁"蜜色橡木、
   与参考图同机位"），标注工作量约每件 2 分钟。

已替换：床、沙发、矮书柜。其余仍是程序化兜底。

## 扩展

**新家具**：`js/sprites.js` 写 `bakeXxx()`（返回 `{surface, offset, w, d, h, front?, emitters?}`，
局部等距原点 = 占格北角），在 `js/engine.js` 的 `CATALOG` 注册交互
（`approach`/`at`/`pose`/`effect`）。按钮、编辑清单、寻路自动跟上；要进日程就往
`SCHEDULE` 加一行。

**新墙饰**：`js/sprites.js` 用 `wallFrame(len, zTop, zBottom, side)` 写烘焙函数，
`engine.bakeWallDecor` 加一个 case，然后在布局 `wallDecor` 里声明
`{type, side: 'NE'|'NW', t}`。返回 `glowPts` 的墙饰会自动参与夜间灯光。

## 文件

| 文件 | 职责 |
| --- | --- |
| `js/pixel.js` | Surface 光栅化内核：扫描线多边形/椭圆、抖动、字符画精灵、调色板 |
| `js/sprites.js` | 家具/地板/地毯/墙饰烘焙（含发光点与前景层） |
| `js/actor.js` | 角色精灵：字符画 + 行走帧生成 + 镜像 |
| `js/engine.js` | 目录、布局、寻路、深度排序、昼夜灯光、日程、编辑器、IO |
| `js/main.js` | 页面接线 |
| `js/audit.js` | 验收画廊 |

本原型独立于 `assets/dashboard/` 与 `src/companion_daemon/`；接入 dashboard 时把
`Engine.dispatch`/`setActor` 映射到 daemon 的 scene state 即可。
